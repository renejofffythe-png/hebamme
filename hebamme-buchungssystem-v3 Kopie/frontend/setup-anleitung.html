Schritt 2 von 8

Node.js installieren

Node.js ist die Software, die dein Backend ausführt. Stell dir vor: Node.js ist der Motor, das Backend ist das Auto.

⬇️ Installation

  1. Geh auf nodejs.org
  2. Klick auf den großen grünen Button „LTS" (nicht „Current")
  3. Installiere es wie jedes normale Programm (einfach auf „Weiter" klicken)

✅ Testen ob es geklappt hat

Öffne das Terminal:

  • Mac: ⌘ + Leertaste → „Terminal" eintippen → Enter
  • Windows: Windows-Taste → „cmd" eintippen → Enter

Tippe diesen Befehl ein und drücke Enter:

node --version
Wenn du etwas wie v22.0.0 siehst – perfekt! Node.js ist installiert. ✅
Schritt 3 von 8

Dateien einrichten

Wir legen den Projektordner an und installieren alle benötigten Bausteine mit einem einzigen Befehl.

📁 Ordner anlegen

  1. Erstelle einen neuen Ordner auf deinem Computer, z.B. auf dem Desktop: hebamme-buchungssystem
  2. Kopiere alle Dateien, die du von mir bekommen hast, in diesen Ordner

💻 Terminal in den Ordner navigieren

Im Terminal (das du vorhin geöffnet hast):

Mac:

cd ~/Desktop/hebamme-buchungssystem

Windows:

cd %USERPROFILE%\Desktop\hebamme-buchungssystem

📦 Bausteine installieren (einmalig, ~2 Minuten)

npm install
Du siehst viele Zeilen Text – das ist normal! Am Ende steht etwas wie „added 200 packages". ✅

📄 .env Datei erstellen

Jetzt erstellen wir die Datei mit deinen geheimen Zugangsdaten:

# Mac/Linux:
cp .env.example .env

# Windows:
copy .env.example .env

Öffne die neue .env Datei mit einem Texteditor (z.B. Notepad). Wir füllen sie in den nächsten Schritten aus.

Schritt 4 von 8

Stripe einrichten

Stripe ist dein Zahlungsdienstleister – wie ein digitales Kartenterminal. Im Testmodus kostet dich das gar nichts.

🔑 Konto erstellen & API-Keys holen

  1. Geh auf stripe.com/de → „Konto erstellen"
  2. Mit E-Mail + Passwort registrieren, E-Mail bestätigen
  3. Du landest im Dashboard. Oben rechts siehst du: „Testmodus" – lass den AN ✅
  4. Links im Menü: Entwickler → API-Schlüssel
  5. Du siehst zwei Schlüssel – beide brauchst du gleich

🔐 Geheimer Schlüssel: beginnt mit sk_test_... → kommt in die .env Datei (niemals teilen!)
📢 Veröffentlichbarer Schlüssel: beginnt mit pk_test_... → kommt ins Frontend (ist ok zu zeigen)

🔗 Webhook einrichten (für zuverlässige Buchungsbestätigung)

  1. Im Stripe Dashboard: Entwickler → Webhooks → Endpunkt hinzufügen
  2. URL eingeben: http://localhost:3000/webhook (für lokalen Test)
  3. Events auswählen: payment_intent.succeeded
  4. Speichern → den „Signaturgeheimnis" (whsec_...) kopieren
⚠️ Für den lokalen Test kannst du den Webhook auch erstmal weglassen. Das System funktioniert trotzdem – der Webhook ist nur eine extra Sicherheit.
Schritt 5 von 8

Gmail App-Passwort

Google lässt Apps nicht mit deinem normalen Passwort auf Gmail zugreifen. Deshalb erstellen wir ein spezielles „App-Passwort" – das ist in 5 Minuten erledigt.

🔐 App-Passwort erstellen

  1. Geh auf myaccount.google.com/security
  2. Scrolle zu „Wie du dich bei Google anmeldest"
  3. Klicke auf „2-Schritt-Verifizierung" → aktivieren (falls noch nicht aktiv)
  4. Dann zurück zur Security-Seite → suche nach „App-Passwörter"
  5. App-Name eingeben: Hebamme Buchung → Erstellen
  6. Du bekommst ein 16-stelliges Passwort angezeigt → sofort kopieren!
⚠️ Das App-Passwort wird nur einmal angezeigt! Kopiere es sofort und sichere es, bevor du das Fenster schließt.

🔄 Alternative ohne Gmail: Du kannst auch jeden anderen E-Mail-Anbieter nutzen. Bei web.de oder GMX findest du die SMTP-Daten in den Kontoeinstellungen unter „E-Mail-Programm einrichten".

Schritt 6 von 8

.env Datei ausfüllen

Jetzt trägst du alle Zugangsdaten in die .env Datei ein. Öffne sie in einem Texteditor (Notepad, TextEdit, etc.).

📝 So sieht die fertig ausgefüllte .env aus:

# Stripe
STRIPE_SECRET_KEY=sk_test_51AbCdEf...
STRIPE_PUBLISHABLE_KEY=pk_test_51AbCdEf...
STRIPE_WEBHOOK_SECRET=whsec_abc123...

# Server
PORT=3000
FRONTEND_URL=http://localhost:3000

# E-Mail (Gmail)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=[email protected]
SMTP_PASS=abcd efgh ijkl mnop <-- App-Passwort

EMAIL_FROM_NAME=Hebamme Kristina Schuldeis
EMAIL_FROM_ADDRESS=[email protected]
ADMIN_EMAIL=[email protected]

⚡ Und dann noch: Stripe Key ins Frontend

Öffne frontend/index.html in einem Texteditor und suche nach dieser Zeile (ca. Zeile 570):

const STRIPE_KEY = 'pk_test_DEIN_PUBLISHABLE_KEY';

Ersetze pk_test_DEIN_PUBLISHABLE_KEY mit deinem echten Publishable Key aus Stripe.

✅ Wenn alle Felder ausgefüllt sind, bist du bereit zum Starten!
Schritt 7 von 8

Backend starten

Jetzt starten wir das Backend! Ab hier siehst du dein Buchungssystem zum ersten Mal live.

🚀 Starten

Im Terminal (du solltest noch im Projektordner sein):

npm start
Du siehst:

🌿 Hebamme Buchungssystem läuft auf http://localhost:3000
Frontend: http://localhost:3000/index.html
Stripe-Modus: 🟡 TEST
Admin-E-Mail: [email protected] ✅

🌐 Website aufrufen

Öffne deinen Browser und geh auf:

http://localhost:3000

Du solltest jetzt deine Website sehen – komplett mit allen Kursen!

⚠️ Das Terminal muss offen bleiben, solange du testest. Wenn du es schließt, stoppt das Backend. Zum Stoppen: Strg+C im Terminal.

📋 Buchung durchführen

  1. Geh auf http://localhost:3000
  2. Klicke bei einem Kurs auf „Jetzt buchen"
  3. Fülle das Formular aus (beliebige Daten, aber echte E-Mail!)
  4. Akzeptiere AGB, Teilnahmebedingungen, Stornierung und Datenschutz
  5. Wähle Kreditkarte und gib die Testkarte ein
  6. Klicke auf „Jetzt buchen & bezahlen"

✅ Was du danach prüfen solltest

🎉 Alles grün?

Dann ist dein Buchungssystem einsatzbereit! Wenn du live gehen möchtest, einfach die Stripe Test-Keys durch Live-Keys ersetzen und auf einem Server wie Render.com deployen.

Bei Problemen: Fehlermeldungen im Terminal lesen – die erklären meist genau was fehlt.
